# Impeccable Design Context

## Maintenance Rule

If the current handoff doc pertains to frontend or visual design, update this file in the same pass so the persistent design context stays aligned with the active branch direction.

## Design Context

### Users

Experienced Discogs collectors with 1,000+ release libraries who need to split, merge, or reorganize collections across accounts. These are power users — technically comfortable, detail-oriented, and deeply invested in their collection's accuracy. They run this tool locally on macOS, deliberately, not casually. They're patient with complexity but unforgiving of confusion.

**Job to be done:** Execute high-stakes collection migrations with confidence — knowing exactly what will move, where it will go, and what happened if something goes wrong.

### Brand Personality

**Bold. Technical. Focused.**

A no-nonsense power tool built for people who know exactly what they want. No hand-holding, no decorative chrome. Dense with capability, rewarding to master. Think professional audio equipment: serious, purposeful, built to last.

### Emotional Goals

The interface should make users feel **powerful and in control** — like operating a professional-grade instrument. They should trust it completely. Confidence, not anxiety, even when performing destructive operations.

### Aesthetic Direction

- **Reference:** Discogs itself — match the aesthetic universe the user already lives in. Music-forward, familiar, slightly editorial. Typography that nods to print and liner notes. Structured but with character.
- **Active branch exploration:** TE-inspired light mode with exposed grid structure. Warm cream background, near-black ink, TE orange accents, darker Discogs-green success/source states, square geometry, and visible table/panel rules.
- **Baseline theme direction:** Dark, but not cyberpunk. Warm darks — navy, near-black with warm undertones. Orange and green accents with editorial typographic hierarchy. The light-grid branch is a deliberate exploration, not a change to the long-term brand unless merged and kept.
- **Feel:** Heavy, intentional, precise. Like a mixing console, technical ledger, or a well-worn record catalog. Dense information presented with clarity.

**Anti-references:**

- NO glowy cyberpunk: no neon accents, no cyan/purple gradients, no sci-fi glow effects
- NO flat minimalist beige: not a calm productivity app with soft shadows and muted neutrals
- NO generic SaaS dashboard patterns
- NO mobile-first consumer app aesthetics

### Design Principles

1. **Density with clarity** — Pack information tightly but never obscure it. Power users want to see everything at once without hunting.
2. **Expose structure, not decoration** — Let borders, dividers, and table rules define the interface. Avoid soft card stacks and cosmetic fills unless the state itself is load-bearing.
3. **Music-first typography** — Use type hierarchy that echoes editorial and print design: strong contrast between display and body, nods to liner notes and catalog indexes. On the active light-grid branch, the working pairing is `IBM Plex Serif` for headings/numerals and `IBM Plex Sans` for dense UI copy so the type feels more machined and square than the previous Bricolage/Instrument stack.
4. **Consequential actions deserve weight** — Destructive operations should feel serious and deliberate, not alarming. Require confirmation and use danger styling when local data or account state can be removed.
5. **Progressive disclosure without hiding power** — Collapsible sections are acceptable when they preserve workflow control: source selection open, destination reference secondary, review as the pre-launch checkpoint, job console always inspectable. Collapse affordance: canvas headers use `.canvas-header-right` to group the meta note + chevron as a unified right block; chevron rests at `--color-rule-strong` and turns TE orange on hover alongside an h2 dim.
6. **Theme can branch, brand cannot** — Dark warmth is the baseline brand direction, but branch-specific explorations such as TE light mode are allowed if they preserve Discogs familiarity, dense precision, and non-generic tooling character.
